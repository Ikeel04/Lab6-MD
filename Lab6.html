Preparación del conjunto de datos

listings$price <- as.numeric(gsub("[$,]", "", listings$price))
listings <- listings %>%
  filter(!is.na(price), price > 0)

p33 <- quantile(listings$price, 0.33, na.rm = TRUE)
p66 <- quantile(listings$price, 0.66, na.rm = TRUE)

listings$precio_categoria <- cut(
  listings$price,
  breaks = c(-Inf, p33, p66, Inf),
  labels = c("barata", "media", "cara")
)

vars_numericas <- c(
  "price",
  "accommodates",
  "bathrooms",
  "bedrooms",
  "beds",
  "minimum_nights",
  "number_of_reviews",
  "review_scores_rating",
  "reviews_per_month",
  "calculated_host_listings_count",
  "availability_365"
)

vars_disponibles <- intersect(vars_numericas, names(listings))
datos <- listings[, c(vars_disponibles, "precio_categoria")]

# Imputamos NAs con la mediana en columnas numéricas
for (col in vars_disponibles) {
  datos[[col]][is.na(datos[[col]])] <- median(datos[[col]], na.rm = TRUE)
}
set.seed(123)
porciento <- 0.70
corte     <- sample(nrow(datos), size = floor(porciento * nrow(datos)))

train <- datos[corte, ]
test  <- datos[-corte, ]

Modelo de regresión KNN para predicción de precio

k_inicial <- round(sqrt(nrow(train)), 0)
cat("Valor inicial de k:", k_inicial, "\n")
## Valor inicial de k: 231
parametros_k <- expand.grid(k = k_inicial)

set.seed(123)
modelo_knn_reg <- train(
  price ~ . - precio_categoria,     
  data       = train,
  method     = "knn",
  preProcess = c("center", "scale"),
  tuneGrid   = parametros_k
)

pred_knn_reg <- predict(modelo_knn_reg, newdata = test)

rmse_knn  <- RMSE(pred_knn_reg, test$price)
mae_knn   <- MAE(pred_knn_reg, test$price)
r2_knn    <- cor(pred_knn_reg, test$price)^2

cat("\n── Resultados del modelo KNN de regresión (k =", k_inicial, ") ──\n")

El modelo KNN de regresión fue entrenado con k = 231 vecinos, valor derivado de la raíz cuadrada del número de observaciones de entrenamiento.

RMSE = 3,742.26: Para propiedades de Airbnb, este margen puede ser significativo en rangos de precio bajo o medio, pero representar un error relativo menor en propiedades de lujo. MAE = 685.01: La mediana del error absoluto es mucho menor que el RMSE, lo que indica que la mayoría de las predicciones están relativamente cerca del precio real, pero existen errores extremos que inflan el RMSE considerablemente. R² = 0.3391: Esto sugiere un ajuste moderado-bajo, indicando que las variables utilizadas no capturan completamente los factores que determinan el precio de una propiedad.

La gráfica muestra que el modelo tiende a predecir precios dentro del rango de 0–15,000, sin importar que el valor real sea mucho mayor. Los valores reales extremos (40,000–50,000) quedan muy por encima de las predicciones, lo que evidencia que KNN con k grande suaviza en exceso las predicciones al promediar demasiados vecinos, perdiendo sensibilidad ante propiedades de alta gama.

Los residuos crecen de forma casi lineal conforme aumenta el precio real, con errores positivos muy elevados en las propiedades más costosas (40,000–50,000). En el rango de precios bajos la distribución de residuos es más simétrica alrededor de cero, lo que indica que el modelo funciona aceptablemente para propiedades de precio típico, que son justamente las que más interesan a SmartStay para sus recomendaciones cotidianas. Sin embargo, para propiedades premium el modelo claramente se subestima el precio, lo que podría llevar a recomendaciones de precio no competitivas en ese segmento.

El KNN, al promediar un gran número de vecinos, tiende a suavizar demasiado las predicciones, lo que le resta capacidad para capturar la variabilidad real de los precios.

listings_clean <- listings %>%
  select(
    price, accommodates, bathrooms, bedrooms, beds,
    minimum_nights, availability_365, number_of_reviews,
    review_scores_rating, reviews_per_month,
    room_type, property_type, instant_bookable

  mutate(

Se seleccionaron las variables predictoras relevantes excluyendo price para evitar data leakage. Con una partición 70/30, se dividió el dataset en entrenamiento y prueba, que garantiza una distribución proporcional de las tres categorías en ambos conjuntos.

train_labels <- train_cls$price_category

Antes de entrenar el modelo KNN, se realizó un preprocesamiento en dos pasos. Primero, se aplicó codificación one-hot a las variables categóricas room_type e instant_bookable, convirtiéndolas en columnas numéricas binarias. Segundo, se normalizaron todas las variables con z-score, usando los parámetros del conjunto de entrenamiento y aplicándolos también al de prueba.

set.seed(123)

8. Modelos con validación cruzada

Para mitigar el sobreajuste identificado, se entrena una versión del modelo de clasificación y otra de regresión usando validación cruzada repetida. El método divide el conjunto de entrenamiento en 10 partes, entrena el modelo en 9 y valida en la restante, rotando la parte de validación, y repite el proceso 3 veces para obtener una estimación más estable del error.

9. Tuneo de hiperparámetros del modelo KNN

En KNN los parámetros tuneables son:

- k: número de vecinos (el principal)

- Métrica de distancia (Euclidiana por defecto en caret)

- Preprocesamiento (centrado y escalado, ya aplicado)

caret con method = “knn” permite tunear k directamente con tuneGrid.
